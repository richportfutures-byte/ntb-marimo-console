# Security Policy

## Supported Versions

Security fixes, if any, will be applied to the current `main` branch only.

## Reporting a Vulnerability

Do not open a public issue for suspected security problems.

If you believe you have found a vulnerability:

1. Contact the repository owner privately through GitHub.
2. Include the affected commit or file paths, reproduction steps, and impact.
3. Keep details private until the issue has been reviewed and a fix or mitigation is available.
